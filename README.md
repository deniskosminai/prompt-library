**Professional Prompt Library** — это системный подход к управлению жизненным циклом промптов (PromptOps). Репозиторий включает в себя библиотеку шаблонов, спроектированных по фреймворку **CLEAR**, систему автоматического тестирования и пайплайны CI/CD.

---

## 🌟 Ключевые особенности

- **CLEAR Framework**: Все промпты оптимизированы для достижения максимальной логичности и краткости.
- **Automated Evaluation**: Автоматическая проверка качества ответов (длина, формат, наличие ключевых слов) с помощью `promptfoo`.
- **Versioning**: Строгое версионирование промптов через Git и мета-данные.
- **CI/CD Ready**: Автоматический запуск тестов при каждом Pull Request через GitHub Actions.
- **Provider Agnostic**: Поддержка OpenAI, Anthropic, Google Gemini и локальных моделей через Ollama.

## 🛠 Быстрый старт (Local Development)

### ПриRequirements:
- Node.js (v18+)
- [Ollama](https://ollama.com/) (для бесплатного локального тестирования)

### Установка:
1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/youruser/my-prompt-library.git
   cd my-prompt-library
   ```
2. Установите инструменты тестирования:
   ```bash
   npm install -g promptfoo
   ```

### Запуск тестов:
Для локального тестирования без использования API-ключей (через Ollama):
```bash
cd tests
promptfoo eval
```

## 📝 Как добавить новый промпт

1. Создайте папку в `/prompts/название-категории/`.
2. Создайте файл `prompt_v1.0.md` и `meta.yaml`.
3. Опишите промпт, используя структуру:
   - **[TASK]**: Четкая задача.
   - **[CONTEXT]**: Ограничения и аудитория.
   - **[INPUT]**: Переменные в формате `{variable}`.
   - **[REFLECTIVE]**: Инструкции по самопроверке для модели.

---

## 🧪 Политика тестирования (Evals)

Мы используем три уровня проверок для каждого промпта:
1. **Deterministic**: Проверка длины (JS) и формата (Regex).
2. **Keyword Match**: Проверка вхождения обязательных SEO-ключей.
3. **Model Grader (Optional)**: Использование более мощной модели для оценки качества (LLM-as-a-judge).

Пример ассерта в `test_cases.json`:
```json
{
  "type": "javascript",
  "value": "output.split(' ').length <= 150",
  "threshold": 1
}
```

## ⚙️ Настройка CI/CD

В репозитории настроен GitHub Action, который:
1. Запускается при любом изменении в папке `prompts/`.
2. Проводит тесты всех измененных промптов.
3. Блокирует Merge, если точность (accuracy) ответов падает ниже установленного порога.

*Для работы в облаке не забудьте добавить `OPENAI_API_KEY` в Settings -> Secrets -> Actions.*
